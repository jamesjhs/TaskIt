TaskIt! Arcade Game Creation and Integration Manual

v1.21.3 · for contributors and AI coding agents

This manual explains how to create a compatible TaskIt! Arcade game module, test it locally, and enable it from Admin > Gamify > Arcade Games. It is written for a human contributor and for an AI agent that may be asked to build or iterate a game using the repository as reference.

1. Mental Model

The TaskIt! app owns the Arcade overlay, achievement unlock rules, daily play budget, token spending, and close/back controls. A game module owns only the content inside #arcadeGameFrame.

When a user opens a game, TaskIt! fetches the enabled arcade catalogue from the server, dynamically loads the configured JavaScript file, looks up the module by gameId, and calls its mount(frame, context) function. When the overlay closes, TaskIt! calls unmount().

Owned by TaskIt!Owned by the game module
Overlay shell, title, subtitle, back button, token balance, daily limit, achievement accessGame UI inside #arcadeGameFrame, game state, scoring, input handling, timers within the game
Script loading and cache busting using the admin catalogue's updatedAtRegistering itself with window.TaskItArcade.register(...)
The game should not navigate away, replace the whole page, edit account data, or assume it is the only JavaScript running on the page.

2. File Placement and Naming

Place game modules in one of these locations:

  • public/js/game-my-game.js
  • public/js/games/my-game.js

The admin file picker lists JavaScript files that either start with game- or live under public/js/games/. Use lowercase, hyphenated filenames. The runtime path selected in Admin should look like /js/game-my-game.js or /js/games/my-game.js.

3. Required Wrapper Contract

Every game file must register exactly one game definition:

(function () {
  'use strict';

  window.TaskItArcade.register({
    gameId: 'my_game',
    mount: function (frame, context) {
      // Build the game UI inside frame.
    },
    unmount: function () {
      // Stop timers and remove global listeners.
    },
    addTime: function (seconds) {
      // Optional. Extend an in-game countdown if the game has one.
    },
  });
}());

Contract Fields

FieldRequiredPurpose
gameIdYesMust match the Game ID entered in Admin. Use lowercase letters, numbers, and underscores only.
mount(frame, context)YesCalled each time the game opens. Clear and populate frame.
unmount()YesCalled when the arcade closes or another game replaces it. Stop intervals, animation frames, audio, and global listeners.
addTime(seconds)NoCalled after a user spends an Arcade Token. Implement only if the game has an internal timer.

4. Starter Template

This is a minimal compatible game. It demonstrates clean mounting, replay, keyboard handling, and teardown.

/**
 * TaskIt! Arcade - My Game
 * Game ID: my_game
 */
(function () {
  'use strict';

  var active = false;
  var score = 0;

  function render(frame) {
    frame.innerHTML = '';

    var root = document.createElement('div');
    root.className = 'my-game-root';
    root.style.cssText = 'display:flex;flex-direction:column;align-items:center;justify-content:center;gap:12px;width:100%;height:100%;padding:16px;text-align:center;';

    var scoreEl = document.createElement('p');
    scoreEl.id = 'my-game-score';
    scoreEl.textContent = 'Score: ' + score;

    var button = document.createElement('button');
    button.className = 'hm-replay';
    button.textContent = 'Score Point';
    button.addEventListener('click', function () {
      score++;
      scoreEl.textContent = 'Score: ' + score;
    });

    root.appendChild(scoreEl);
    root.appendChild(button);
    frame.appendChild(root);
  }

  function onKeyDown(event) {
    if (!active) return;
    if (event.key === ' ') {
      event.preventDefault();
      var button = document.querySelector('#arcadeGameFrame button');
      if (button) button.click();
    }
  }

  window.TaskItArcade.register({
    gameId: 'my_game',
    mount: function (frame) {
      active = true;
      score = 0;
      render(frame);
      document.addEventListener('keydown', onKeyDown);
    },
    unmount: function () {
      active = false;
      document.removeEventListener('keydown', onKeyDown);
    },
  });
}());

5. Design and UX Constraints

  • Render only inside the provided frame. Do not append game UI directly to document.body except for temporary browser-native APIs that are cleaned up.
  • Keep the game responsive from narrow mobile screens to desktop. Use width:100%, height:100%, flex/grid, and stable dimensions.
  • Do not cover the Arcade top bar, token display, back button, or daily limit row.
  • Use existing Arcade CSS classes where sensible. Current games use prefixes such as hm-, ws-, wab-, and cb-. New games should use their own unique prefix.
  • Use buttons for actions and make keyboard/touch interaction work. Escape is reserved by TaskIt! to close the overlay.
  • Do not use network calls unless the game explicitly needs them and the endpoint already exists.
  • Do not store sensitive data in localStorage. Small local preferences or high scores are acceptable only if they are non-sensitive.
  • Avoid large assets. If assets are needed, place them under a dedicated public folder and keep load failures graceful.
  • If using timers, pause/stop them in unmount(). If using requestAnimationFrame, cancel it in unmount().
  • If the game uses audio, default to muted or user-initiated playback and stop it in unmount().
Never depend on DOMContentLoaded to start the game. Game scripts are loaded lazily after the page is already running.

6. Local Testing Checklist

  1. Run the app locally with npm run dev from the repository root.
  2. Add the game file under public/js/game-my-game.js or public/js/games/my-game.js.
  3. Sign in as an admin.
  4. Open Admin > Gamify > Arcade Games.
  5. Click Refresh in the Arcade Games card so the file picker rescans available scripts.
  6. Create or edit a game entry. Set Game ID to the exact gameId from the module.
  7. Select the script file and tick Enabled for users.
  8. Save the entry, then click Test.
  9. Close the overlay and reopen it several times. Confirm there are no duplicate keyboard events, duplicate intervals, stale UI, or console errors.
  10. Test on a narrow mobile viewport and a desktop viewport.
  11. If the game has a timer, spend an Arcade Token and confirm addTime(seconds) behaves correctly.

8. Brief for an AI Agent

Use this prompt when asking an AI coding agent to create a game:

You are adding a TaskIt! Arcade game module.

Read /public/arcade-game-guide.html and inspect the existing game modules in /public/js.
Create a single compatible JavaScript file under /public/js/games/.
Use window.TaskItArcade.register({ gameId, mount, unmount, addTime? }).
Render only inside the frame passed to mount().
Do not depend on DOMContentLoaded.
Clean up all timers, requestAnimationFrame loops, audio, and document/window listeners in unmount().
Use a unique CSS class prefix and either inline scoped styles inside the game root or existing Arcade styles.
Make it responsive for mobile and desktop.
Preserve Escape behavior for closing the Arcade.
After implementation, explain the Admin Arcade Games values needed to register it.

9. Pull Request Review Checklist

  • The file is in public/js/games/ or starts with game-.
  • The module calls window.TaskItArcade.register exactly once.
  • The admin gameId and module gameId match.
  • mount() clears and rebuilds the frame cleanly.
  • unmount() removes global listeners and stops all timers/animation loops.
  • The game remains usable at mobile width and does not overlap the Arcade shell.
  • No broad global CSS selectors are introduced.
  • No unrelated app files are changed unless required.
  • The game has been tested through the Admin Test button.

10. Troubleshooting

SymptomLikely causeFix
File does not appear in Script File dropdownFilename/path does not match scanner rulesUse public/js/game-name.js or public/js/games/name.js, then click Refresh.
Overlay shows placeholder instead of gameScript did not register, or gameId mismatchCheck browser console and confirm Admin Game ID equals module gameId.
Game works once but breaks after reopeningState or listeners are not resetReset state in mount() and clean up in unmount().
Keyboard controls fire multiple timesDocument listener added repeatedly and not removedUse a named listener function and remove it in unmount().
Timer continues after closing ArcadeInterval or animation loop not clearedStore interval/frame IDs and clear/cancel them in unmount().
Updated game file does not appear to changeBrowser/CDN cacheSave the admin game entry again so updatedAt changes and the loader appends a fresh version query.
